Distributed File Storage System (12/2023 – 02/2024)

Developed a high-availability, strong-consistency distributed file storage system with impressive performance.

  • Achieved 20,000 QPS with P99 latency of 800ms for 4KB key-value mixed read/write operations during stress testing.
  • Created a file client to wrap file content into key-value requests with zlib compression.
  • Implemented Raft consensus algorithm: Leader election, Log replication, Snapshot updates.
  • Used consistent hashing for shard partitioning and migration across Raft groups.
  • Supported various storage engines like RocksDB, B-tree, and Hash tables.
  • Optimized I/O models for better performance.
  • Improved read speeds with Asynchronous Apply, ReadIndex, and FollowerRead.
  • Prevented frequent master switches with Prevote mechanism.
